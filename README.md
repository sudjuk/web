## Лабораторная 1

### Цель работы
Выбор варианта/темы на весь курс, знакомство с разработкой бэкенда и разработка дизайна для 3 страниц.

### Порядок показа
1. Показать задание по вашей теме/варианту (см. список внизу).
2. Показать дизайн 3 страниц в Figma и соответствие этому дизайну в приложении.
3. Показать сайт-источник дизайна, перечислить, что было скопировано, и указать это в CSS.
4. Показать три страницы приложения с поиском. Во вкладке Network для 3 запросов GET объяснить URL:
   - параметр фильтрации/поиска,
   - id заявки,
   - id услуг.
5. Для страницы списка услуг в HTML из Response показать:
   - id заявки,
   - количество в корзине,
   - URL изображений из Minio.
6. В коде показать:
   - где в коллекции услуг хранятся эти URL,
   - их использование в 3 шаблонах.
7. Показать роутинг: 3 URL и 3 контроллера, объяснить реализацию фильтрации/поиска.

### Контрольные вопросы
MVT, компоненты MVC в лабораторной, Django, шаблонизация, HTTP и модель OSI, Web, HTML.

### Задание
Создание дизайна приложения в Figma, базовая шаблонизация в Django (для Go — просто HTML) для услуг, развертывание Minio.

### Требования к дизайну
Требуется разработать дизайн трех страниц приложения в Figma, повторив основную стилистику популярного ресурса по вашей теме:
- 3 основных цвета с точными кодами;
- эффекты наведения (hover);
- форма карточек;
- использовать реальные данные по теме.

### Страницы приложения
1. Список услуг (отели, товары, рейсы и т.д.) в виде карточек (плиткой в 2–4 столбца) с:
   - наименованием,
   - ценой/датой/мощностью и т.п.,
   - картинкой.
   Отдельная карточка текущей заявки (корзина) отображает количество услуг в ней (вычисляется в обработчике контроллера).
2. Детальная страница услуги: даты, описание и прочее. Переход по клику на карточку.
3. Страница просмотра состава заявки по её id:
   - поля самой заявки,
   - каждая услуга — отдельной карточкой вертикально,
   - поля услуги выводятся слева направо в одну строку,
   - справа — поле м-м (количество/порядок/главный/комментарий/другое), изменяемое в следующих лабораторных,
   - в заявке или отдельном поле м-м есть поле результата вычислений (пока фиксированное значение).

### Навигация и поиск
- В header добавить только кнопку/иконку «Домой» для перехода на страницу услуг.
- Добавить input для серверной фильтрации списка услуг по одному из отображаемых полей (наименование, цена, дата события).
- Поле поиска должно сохраняться после запроса.

### Данные и ограничения
- В приложении — 3 GET‑запроса и 2 модели/коллекции.
- Данные брать напрямую из двух коллекций (массив услуг и словарь заявки), без БД.
- Внутри отдельной услуги не использовать массивы (атомарность, 1НФ). Большие текстовые описания — допускаются и приветствуются.
- Никакого редактирования: только поиск и просмотр.
- Без JavaScript.

### Стили и ресурсы
- Использовать стили оформления; для каждого элемента списка подгружается своё изображение.
- Изображения услуг отображаются на всех трёх страницах.
- Изображения хранятся в Minio; имя файла — ключ на латинице, хранится в модели отдельным полем.
- Разработать общий стиль приложения, который будет использоваться в дальнейших лабораторных по фронтенду.
- CSS вынести в отдельный файл.
